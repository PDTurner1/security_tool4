It's built, tested, and ready — pure standard library, no dependencies to install.

How it works: you feed it .eml files (exportable from Gmail, Outlook, Apple Mail, etc.), and it runs 10 explainable checks — SPF/DKIM/DMARC results, sender/display-name spoofing, lookalike domains, link-text-vs-actual-URL mismatches, IP-literal links, urgency language, credential-harvesting phrases, dangerous attachments, and more. Each finding carries a point value and a plain-English reason, so nothing is a black box — you can see exactly why an email got flagged.

Quick test against my two sample emails:

A normal newsletter scored 0/100 — LIKELY SAFE

Usage:

bash
python -m phishing_scanner scan path/to/email.eml
python -m phishing_scanner scan path/to/folder/ --csv report.csv

A few honest caveats worth keeping in mind: it's a heuristic scorer, not an ML classifier, so it'll have both false positives (aggressive marketing copy) and false negatives (well-crafted phishing that avoids these patterns) — treat the score as one signal, not a verdict. It also doesn't fetch links or open attachments (by design, for safety), so it can't check things like domain age or scan attachment contents.

The README covers extending it further — adding an allowlist for your org's domains, wiring in a real WHOIS lookup, or eventually feeding the extracted features into a trained classifier if you want to go that route.
